Утилита для помощи системному администратору. Позволяет отправлять сообщения пользователям. Отправленное сообщение сразу появляется на экране пользователя.
Основные возможности:
1.Для администратора доступно окно, в котором он может набирать текст, выбирать получателей сообщения, а также выбирать уровень важности сообщения
2. При отправке сообщения, в случае успешной  доставки у клиента на экране появляется с текстом, введенным администратором.
3. Администратом может видеть, какие узлы в данный момент доступны и какие нет.
4. Администратор может группировать узлы (отдел закупок, бухгалтерия...) и отправлять сообщения на группы
5. Администратор может видеть список отправленных сообщений, а также статус доставки по каждому адресату (не доставлено, доставлено, прочитано)
6. В случае получения клиентом сообщения статус меняется на "доставлено". В случае нажатия на "ОК" статус меняется на "прочитано".

Приложения
Приложение администратора - для посылки сообщений
Приложение клиента - для приёма сообщений

Окна
1. Приложение администратора
	1. Окно "Новое сообщение". Позволяет написать новое сообщение группе пользователей.
	2. Окно "Журнал сообщений". Показывает журнал отправленных сообщений. Текст сообщения, когда отправлено, кому отправлено. Результат отправки каждому получателю.
	3. Окно "Настройки". всякие настройки.
	
2. Прилождение клиента
	1. Окно "Сообщение" - появляется, когда клиент получает сообщение от Администратора
	2. Окно "Настройки" - настройка внешнего вида сообщений и прочего.
	
Хранение данных.
	Данные об отправленных сообщениях хранятся на компьютере Администратора в файле.
	Данные о настройках Приложения Администратора хранятся на компьютере Администратора в файле.
	Данные о настройках Приложения Клиента хранятся на компьютере Клиента в файле.

	Алгоритмы
0. Ожидание сообщения от клиента
	1. При запуске программы запускается UDP (?) клиент, который слушает входящие сообщения на определенном порту.
	2. Клиент по необходимости отсылает сообщения на этот порт (сообщение доставлено, сообщение прочитано).
	3. Программа анализирует эти сообщения и в соответствии с ними меняет определенные данные.
	
1. Отправка сообщения.
	1. Пользователь открывает окно сообщения.
	2. Программа формирует список получателей. Отображает, какие из них доступны.
		2.1. Формирование списка доступных происходит путем отправки сначала пинга, а затем специального запроса. Если вернулся пинг и ответ на запрос - значит, всё в порядке. Если вернулся пинг, а ответ на запрос не вернулся- значит, не запущен клиент. Если не вернулся пинг - значит, узел выключен.
	3. Пользователь вводит текст, выбирает получателей. Нажимает "Отправить"
	4. Программа формирует список получателей сообщения.
	5. Для каждого получателя отправляется сообщение.

2. Приём сообщения 
	1. Клиент слушает порт на предмет сообщения.
	2. Если сообщение - пинг-запрос, то клиент отправляет ответ на него.
	3. Если сообщение - это текстовое сообщение, то клиент выдает его пользователю и отправляет ответ, что сообщение получено.
	
3. Отображение сообщения пользователю
	1. Клиент отображает окно с сообщением пользователю.
	2. Если пользователь нажимает ОК, клиент отправляет Администратору сообщение, что сообщение прочитано.
	
Формат сообщений
[ID СООБЩЕНИЯ]|[ТИП СООБЩЕНИЯ]|[ТЕКСТ СООБЩЕНИЯ]|CRC
ID - 8 символов 0..F (16^8 = 2^32 вариантов) A05F6FBC
ТИП СООБЩЕНИЯ - 4 символа. 
	PING - пинг - запрос
	PRPL - пинг-ответ
	MESG - сообщение от сервера
	RCVD - сообщение получено
	READ - сообщение прочитано
	RTRY - ошибка при приёме сообщения, нужно послать еще раз.
ТЕКСТ СООБЩЕНИЯ - содержимое сообщения. зависит от типа сообщения	
	для PING - пусто
	для PRPL - ID PING-запроса
	для MESG - текст сообщения администратора
	для RCVD - ID полученного сообщения
	для READ - ID прочитанного сообщения
	для RTRY - ID сообщения, которое требуется послать заново
CRC - контрольная сумма сообщения. 8 символов.

списки ID сообщений у сервера и клиентов раздельны и не связаны.
Сценарии обмена:

1. ПИНГ-запрос и ответ
АДМИН: 0000AB43|PING||133E223B
ЮЗЕР: 00000056|RPLY|0000AB43|D4C78A89

2. отправка, получение и чтение сообщения
АДМИН: 0000AB44|MESG|Тестовое сообщение|D4F0361A
ЮЗЕР: 00000057|RCVD|0000AB44|BF6ECB9A
ЮЗЕР: 00000058|READ|0000AB44|E9E1770E

3. ошибка при приёме сообщения

АДМИН: 0000AB44|MESG|Тестовое сообщение|D4F0361A
ЮЗЕР: 00000057|RTRY|0000AB44|107ED9AB
АДМИН: 0000AB44|MESG|Тестовое сообщение|D4F0361A
ЮЗЕР: 00000057|RCVD|0000AB44|BF6ECB9A
ЮЗЕР: 00000058|READ|0000AB44|E9E1770E
